# IdeaForge — Thinking Methods (shared toolbox)
*Concrete methods each thinker can draw on so outputs are substantive, not generic. Use the ones that fit the idea; name the method when it helps the user follow the reasoning.*

## Framing — Philosopher
- **First principles** — strip to what's undeniably true, rebuild from there.
- **Five Whys** — chase the problem behind the problem.
- **Jobs-to-be-Done** — what job is this idea hired to do?
- **Problem reframing** — restate the problem >=2 other ways; the best ideas often come from a better question.
- **Define success first** — name the metric before generating, or you optimize the wrong thing.

## Divergence — Explorer, Synthesist
- **SCAMPER** — Substitute, Combine, Adapt, Modify, Put-to-other-use, Eliminate, Reverse.
- **Inversion** — "how would we guarantee the opposite / make this fail?" then flip it.
- **Analogy & biomimicry** — how is this solved in nature, other industries, history, games?
- **Random stimulus** — force a link to an unrelated concept.
- **10x and 10%** — what would this look like 10x bigger, or 10x cheaper/smaller?
- **Combinatorial** — merge two weak ideas into one strong one.
- Rule: **quantity first, judgment later.** No critiquing during divergence.

## Demand — Advocate
- **JTBD + struggling moment** — the real need shows up at the point of struggle.
- **Vitamin vs painkiller** — nice-to-have or must-solve?
- **Early-adopter test** — who needs this so badly they'd use a rough version tomorrow?

## Feasibility & systems — Realist
- **What-would-have-to-be-true** — list the load-bearing assumptions.
- **Second-order thinking** — "and then what?" two steps out.
- **Constraints map** — money, time, skill, tech, legal, attention.
- **Theory of constraints** — find the one bottleneck that governs the rest.

## Critique — Skeptic
- **Premortem** — assume failure, explain it backward.
- **Red team** — argue the opponent's or competitor's best move.
- **Outside view / base rates** — how often does this reference class succeed?
- **Bias checklist** — confirmation, sunk cost, optimism, availability, authority, novelty.
- **Steelman first** — never attack a weak version.

## Foresight — Futurist
- **STEEP / PESTEL** — Social, Technological, Economic, Environmental, Political forces.
- **Three Horizons** — H1 now, H2 transition, H3 emerging.
- **2x2 scenarios** — pick the two biggest uncertainties, build four futures.
- **Wildcards / black swans** — low-probability, high-impact shocks.
- **Backcasting** — start from the desired future, work back to today.

## Convergence — Strategist
- **Impact x Confidence x Effort (ICE)** — fast triage score.
- **Desirability / Feasibility / Viability** — all three must hold.
- **Weighted criteria** — score options against the FRAME's success definition.
- **Reversible vs one-way-door** — bias to action on reversible calls; slow down on irreversible ones.
- **Dot-voting** — surface the team's collective lean, then test it rather than obey it.
