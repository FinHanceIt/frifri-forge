# Measurement

No protocol ships without measures and kill criteria. This is the studio's non-negotiable discipline: a self-transformation protocol without a way to know whether it's working, or a rule for when to stop, is not a protocol — it's a hope.

## 1. Core Principle: Placebo-Honest Measurement

Measurement itself changes behavior — this is **reactivity**, a well-documented phenomenon where the act of observing/tracking a behavior shifts it, independent of any intervention. This studio uses reactivity deliberately (a daily log is itself a mild behavior-change lever) **and names it plainly** to the user rather than pretending all improvement came from the protocol's "active ingredient." This is the same honesty standard applied to placebo/expectation effects elsewhere in this studio (see evidence-map.md §13) — expectation and self-observation are real, legitimate levers, not tricks, but they must be labeled as what they are.

## 2. Instruments

- **SUDs (Subjective Units of Distress), 0–10**: originally introduced by Wolpe & Lazarus (1966) as a 0–100 scale within systematic desensitization; this studio uses the now-common condensed **0–10** version for ease of daily use. Self-rated, momentary distress/anxiety intensity. Use before/after a targeted exposure, trigger, or protocol session.
- **Confidence / craving / belief-strength ratings, 0–10**: same self-report logic as SUDs, applied to whichever construct the protocol targets (e.g., "how strong was the urge," "how much do you believe [identity statement] right now").
- **Frequency counts**: raw tallies of the target behavior (times performed / times avoided / times substituted) — the simplest, least-inference-dependent measure available and the default primary outcome when in doubt.
- **Time-to-recovery after trigger**: minutes/hours from a triggering event to return to a self-defined baseline state — useful for protocols where the goal is resilience/recovery speed rather than trigger elimination.
- **Weekly behavioral evidence log**: a running list answering "what did I do this week that the old pattern would have prevented?" — directly operationalizes McAdams's redemption-sequence framing (narrative-methods.md §1) and Clear's identity-votes framing (habit-engineering.md §3) into a trackable weekly artifact, not just a feeling.

## 3. Self-Experiment Design

- **Baseline week**: always measure for one full week *before* starting any protocol change. Without a baseline, "improvement" cannot be distinguished from normal week-to-week variation, regression to the mean, or the fresh-start effect alone (habit-engineering.md §6).
- **One primary outcome, max 2 secondary**: pick the single measure that most directly answers "did this work," and at most two supporting measures. More than that and the self-experiment collects data faster than it can be interpreted — a classic self-tracking failure mode.
- **ABAB logic where feasible**: baseline (A) → protocol (B) → brief return to baseline (A) → protocol again (B). If the target measure tracks the on/off pattern, that's much stronger evidence of a real effect than a single before/after comparison, which is confounded with time, fresh-start effects, and reactivity alone. Not always feasible or ethical (e.g., don't deliberately return to a harmful baseline) — use judgment per protocol.
- **Expectation effects acknowledged**: name upfront that some of any observed improvement is very likely expectation/placebo/reactivity, not only the specific mechanism the protocol claims to use. This is honest framing, not a discouragement — expectation is a real, usable lever (see §1).
- **Journaling as data**: qualitative notes (what happened, what it felt like, what surprised you) are data, not filler — read them back at the weekly review, not just the numbers.

## 4. Review Cadence

- **Daily**: 1-minute log — the frequency count / SUDs rating / one-line note. Must be fast enough to survive a busy day, or it gets abandoned and the whole self-experiment goes dark.
- **Weekly**: 15-minute review using this template:
  1. Adherence % (days protocol was actually followed / days planned)
  2. Primary + secondary measure trend (better / same / worse, with the actual numbers)
  3. One thing that surprised you this week (from the journal)
  4. Adjust exactly **one variable** for next week — never more than one, or you won't know what caused any change
- **28-day verdict**: at the 4-week mark, issue one of three calls:
  - **Continue** — measures trending the right direction, adherence reasonable → keep going, possibly to the next milestone (see habit-engineering.md §8 for realistic automaticity timelines — 28 days is often still early).
  - **Adjust** — flat or mixed trend → change the one variable the weekly reviews flagged as most likely responsible, then run another cycle.
  - **Stop** — see kill criteria below; this is not a failure state, it's the system working as designed.

## 5. Kill Criteria (Mandatory in Every Protocol)

A protocol is not allowed to ship without an explicit, named stop condition. Any of the following → **stop the protocol and consider professional support** (route per safety-boundaries.md):

- Measures worsening for **2+ consecutive weeks** (not one bad week — trend, not noise)
- Distress (SUDs or equivalent) increasing rather than decreasing over the protocol's stated timeframe
- Functioning declining — work, relationships, or daily responsibilities getting measurably harder to manage
- Sleep collapsing — this is often the earliest and most reliable warning sign of an intervention (especially introspective or intensive ones) doing more harm than good, and should be treated as an independent kill switch even if other measures look fine

None of these criteria require the user to justify stopping, defend the decision, or "push through" — a triggered kill criterion is a designed exit, not a failure of willpower, and should be framed that way every time it fires.
